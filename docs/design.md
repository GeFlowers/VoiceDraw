# Voice Draw 设计文档

## 目标

开发一款纯语音控制的 AI 绘图工具。用户不依赖鼠标或键盘，通过自然语言语音完成绘图创作。系统需要兼顾三点：

- 指令理解准确性与容错性
- 语音到绘图操作的响应延迟
- 复杂指令的拆解与执行能力

## 总体架构

```text
浏览器语音识别
    |
    v
FastAPI /api/commands/interpret
    |
    v
LangGraph 指令理解工作流
    ├── normalize: 文本归一化
    ├── rule_parse: 本地规则解析
    ├── llm_parse: LangChain + OpenAI 兼容接口兜底
    └── validate: 操作计划校验与修复
    |
    v
CommandPlan JSON
    |
    v
前端 Canvas 执行绘图操作
```

## 指令处理策略

### 低延迟路径

常见指令优先使用本地规则解析，不等待 LLM：

- 基础图形绘制
- 颜色、线宽、填充、透明度
- 位置锚点
- 选中、删除、移动、缩放、旋转
- 撤销、重做、清空、背景、导出
- 简单场景预设

这条路径适合高频绘图操作，响应延迟主要来自浏览器语音识别和一次本地 API 请求。

### AI 兜底路径

当规则解析置信度低于 `RULE_CONFIDENCE_THRESHOLD` 且 `.env` 中配置了 `LLM_BASE_URL` 和 `LLM_API_KEY`，LangGraph 会进入 LLM 节点。LLM 节点使用 LangChain 的 `ChatOpenAI` 兼容接口，将自然语言转为严格的 `CommandPlan` JSON。

如果 LLM 调用失败，工作流不会中断，会回退到规则解析结果，并把失败原因写入 `warnings`。

| 能力 | 计划说明 |

| 画布操作 | 清空、背景色、透明背景、导出 PNG |

| 容错反馈 | 无法理解时不执行危险操作，并通过语音/界面反馈 |

## 准确性设计

- 文本归一化：统一全角/半角字符、大小写、标点。
- 规则优先：高频命令使用确定性解析，减少模型幻觉。
- Pydantic 协议：所有后端输出必须符合 `CommandPlan`。
- 统一校验：缺少样式或几何时补默认值；缺少 shape 的绘图动作会被跳过。
- 保守执行：无法理解时返回 `no_op`，不会误删或误清空。
- 目标推断：有选中对象时默认操作选中对象，否则操作最后一个对象。

## 复杂指令拆解

规则解析会按以下连接词拆解命令：

- `然后`
- `接着`
- `之后`
- `随后`
- `并且`
- `同时`
- `另外`
- `再画`
- `再写`
- `还有`
- `以及`
- 中文和英文逗号、句号、分号

示例：

```text
画一个红色圆，然后在右边画蓝色矩形，再在顶部写上标题 Voice Draw
```

会拆成三个操作：

1. 绘制红色圆
2. 在右边绘制蓝色矩形
3. 在顶部添加文字

## 响应延迟设计

- 前端使用队列串行处理语音结果，避免多个语音片段并发修改画布。
- 后端规则解析不调用外部服务，适合高频命令。
- LLM 仅在低置信度时触发，且受 `LLM_TIMEOUT_SECONDS` 控制。
- 复合指令在一次请求内返回完整操作计划，前端一次性应用，减少多次往返。

## 模块职责

| 模块 | 职责 |
| --- | --- |
| `backend/app/domain/models.py` | 指令请求、绘图操作、样式、几何和计划模型 |
| `backend/app/services/text_normalizer.py` | 文本归一化、命令拆分、数字解析 |
| `backend/app/services/colors.py` | 颜色、线宽、透明度、填充解析 |
| `backend/app/services/rule_parser.py` | 本地规则解析和场景预设 |
| `backend/app/services/llm_parser.py` | LangChain 模型调用和结构化输出 |
| `backend/app/services/interpreter_graph.py` | LangGraph 工作流编排 |
| `backend/app/services/validators.py` | 操作计划校验和修复 |
| `backend/app/static/app.js` | 语音识别、API 调用、Canvas 执行、历史栈 |

## 后续扩展方向

- 为对象增加可检索属性索引，支持“选择左边的红色圆”。
- 增加会话记忆节点，让复杂多轮编辑能引用更自然的上下文。
- 接入图像生成模型，将“画一只写实猫”转换为位图图层。
- 增加 WebSocket 流式反馈，进一步降低用户等待感。
- 增加端到端浏览器测试，覆盖真实语音识别以外的前端执行路径。
