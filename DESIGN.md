# VoiceDraw 设计文档

## 项目概述

VoiceDraw 是一个基于中文语音输入的 AI 矢量绘图工具。用户描述想要绘制或编辑的内容，后端调用 OpenAI-compatible AI API，将语音识别文本转换为结构化绘图计划，前端再根据计划在 HTML Canvas 上执行绘制和编辑。

项目当前采用 AI-only 规划方式：所有绘图命令都交给 AI 解析，不再保留本地规则兜底逻辑。

## 当前目录结构

```text
VoiceDraw/
  backend/
    config.py
    main.py
    models.py
    llm_parser.py
    validators.py
  fronted/
    index.html
    app.js
    styles.css
  requirements.txt
  README.md
  DESIGN.md
```

说明：`fronted` 是当前前端目录名。后端会优先寻找 `voicedraw/fronted`，不存在时回退到根目录下的 `fronted`。

## 已支持功能

### 1. 前端页面托管

后端通过 `backend/main.py` 提供静态文件服务：

- `GET /` 返回前端页面
- `GET /app.js` 返回前端逻辑
- `GET /styles.css` 返回前端样式

前端文件位于：

```text
fronted/
```

### 2. AI 配置读取

`backend/config.py` 负责读取配置。

支持从 `.env` 或系统环境变量读取：

- `APP_ENV`
- `LLM_BASE_URL`
- `LLM_API_KEY`
- `LLM_MODEL`
- `ENABLE_LLM`
- `LLM_TIMEOUT_SECONDS`

后端通过 `llm_ready` 判断 AI 是否可用。

### 3. 健康检查接口

接口：

```http
GET /api/health
```

返回内容包括：

- 服务状态
- 当前环境
- AI 是否已配置
- 当前模型名称
- 当前规划器模式

### 4. 语音命令解释接口

接口：

```http
POST /api/commands/interpret
```

前端会提交：

- 用户语音识别文本
- 当前画布宽高
- 当前选中对象
- 最近对象 ID

后端返回 `CommandPlan`，前端根据其中的 `operations` 执行绘图或编辑。

### 5. AI 枚举化绘图规划

`backend/llm_parser.py` 会把可用枚举提供给 AI，要求 AI 先总结用户意图，再生成操作。

当前提供给 AI 的枚举包括：

- `OperationType`
- `ShapeType`
- `TargetSelector`
- `PathCommand`

AI 需要从这些固定枚举里选择动作类型、图形类型、目标对象和路径命令，避免生成前端无法识别的动作。

### 6. 支持的操作类型

当前支持的操作包括：

- `draw_shape`：绘制基础图形
- `draw_path`：绘制自由矢量路径
- `add_text`：添加文字
- `set_style`：修改样式
- `set_background`：修改背景
- `set_canvas_size`：修改画布尺寸
- `select`：选择对象
- `delete`：删除对象
- `move`：移动对象
- `resize`：缩放对象
- `rotate`：旋转对象
- `clear`：清空画布
- `undo`：撤销
- `redo`：重做
- `export`：导出图片
- `announce`：仅提示，不改变画布
- `no_op`：无操作

### 7. 支持的基础图形

当前支持的图形包括：

- 路径：`path`
- 线条：`line`
- 箭头：`arrow`
- 矩形：`rectangle`
- 圆形：`circle`
- 椭圆：`ellipse`
- 三角形：`triangle`
- 菱形：`diamond`
- 五边形：`pentagon`
- 六边形：`hexagon`
- 星形：`star`
- 爱心：`heart`
- 花朵：`flower`
- 云朵：`cloud`
- 太阳：`sun`
- 树：`tree`
- 房子：`house`
- 山：`mountain`
- 笑脸：`smile`
- 闪电：`lightning`
- 文字：`text`

### 8. 自由路径绘制

复杂图形可以通过 `draw_path` 绘制。

路径命令支持：

- `M`：移动到某点
- `L`：画直线到某点
- `Q`：二次贝塞尔曲线
- `C`：三次贝塞尔曲线
- `Z`：闭合路径

坐标统一使用 `0` 到 `1` 的归一化画布坐标。

### 9. 前端 Canvas 绘制

前端使用 HTML Canvas 实现绘制。

支持：

- 绘制基础图形
- 绘制路径
- 绘制文字
- 选中对象
- 移动对象
- 缩放对象
- 旋转对象
- 删除对象
- 清空画布
- 撤销/重做
- 导出 PNG
- 修改画布尺寸
- 修改背景色

### 10. 语音输入

前端使用浏览器 Web Speech API 获取语音识别结果。

当前支持：

- 中文语音识别
- 实时显示临时识别文本
- 最终语音文本提交给后端 AI

当前不进行语音朗读反馈。

### 11. AI 调用状态展示

前端显示：

- AI 连接状态
- 当前模型
- AI 延时
- 操作数
- AI 置信度
- 历史记录数

当前不显示“来源”，因为系统已经改为 AI-only。

### 12. 返回结果校验

`backend/validators.py` 会校验 AI 返回的绘图计划。

当前校验规则包括：

- `draw_shape` 必须有 `shape`
- `draw_shape` 和 `add_text` 必须有 `geometry`
- `draw_path` 必须有 `path`
- `set_canvas_size` 必须有 `value`
- 变换类操作没有指定目标时默认作用于 `selected`

校验器不会再本地生成默认图形，只会过滤不完整或非法的 AI 操作。

## 当前执行流程

### 页面访问流程

```text
浏览器访问 /
  -> backend/main.py
  -> 查找前端目录
  -> 返回 index.html
```

### 命令执行流程

```text
用户语音输入
  -> fronted/app.js
  -> POST /api/commands/interpret
  -> backend/main.py
  -> CommandRequest
  -> LLMCommandParser
  -> AI API
  -> CommandPlan
  -> PlanValidator
  -> JSON 返回前端
  -> Canvas 执行 operations
```

## 想实现但当前未实现的功能

### 1. 对象持久化保存

未实现内容：

- 保存画布项目文件
- 重新打开历史项目
- 持久化对象列表和编辑历史

未实现原因：

当前项目没有数据库、文件存储或用户系统。前端对象状态只保存在浏览器运行时内存中，刷新页面后会丢失。

### 2. 多用户和账号系统

未实现内容：

- 用户登录
- 用户项目隔离
- 云端同步
- 权限管理

未实现原因：

当前目标是单用户本地演示工具，后端没有认证、会话、数据库和用户模型。

### 3. 精确对象选择

未实现内容：

- 鼠标点击选择单个对象
- 框选多个对象
- 图层面板选择对象

未实现原因：

当前主要交互入口是语音和文字命令，前端只维护基础选中状态，没有完整的命中测试、图层管理和多选交互系统。

### 4. 图层系统

未实现内容：

- 上移一层
- 下移一层
- 置顶
- 置底
- 锁定图层
- 隐藏图层

未实现原因：

当前对象只是按数组顺序绘制，没有抽象出独立图层模型。要实现图层系统，需要扩展对象数据结构和前端操作面板。

### 5. 更完整的样式编辑

未实现内容：

- 渐变填充
- 阴影
- 端点样式
- 线帽样式
- 字体选择
- 字号独立控制

未实现原因：

当前样式协议只包含 `stroke`、`fill`、`line_width`、`opacity`、`dashed`。复杂样式需要扩展后端模型、AI 输出协议和 Canvas 渲染逻辑。

### 6. AI 结果可视化确认

未实现内容：

- AI 先预览计划
- 用户确认后再绘制
- 展示 AI 拆解出的每个操作
- 用户手动删除某个操作

未实现原因：

当前交互路径是“AI 返回后立即执行”，没有中间确认态。实现该功能需要前端增加计划预览 UI 和操作队列管理。

### 7. AI 失败重试和模型降级

未实现内容：

- 自动重试
- 多模型 fallback
- 超时后切换备用模型
- 更细粒度错误提示

未实现原因：

当前后端只配置一个 OpenAI-compatible endpoint 和一个模型。系统已经移除本地规则兜底，因此 AI 不可用时直接报错。

### 8. 流式 AI 反馈

未实现内容：

- AI 边生成边显示
- 分步骤绘制
- 实时显示规划过程

未实现原因：

当前使用普通 `/chat/completions` JSON 响应，一次性等待完整结果。要实现流式反馈，需要改造 HTTP 响应方式和前端状态处理。

### 9. 更强的路径编辑能力

未实现内容：

- 拖动路径控制点
- 编辑贝塞尔曲线
- 路径节点增删
- 路径平滑

未实现原因：

当前 `draw_path` 主要由 AI 一次性生成，前端只负责渲染，不提供矢量编辑器级别的节点操作。

### 10. 真实语音识别准确率展示

未实现内容：

- 显示浏览器语音识别置信度
- 区分语音识别置信度和 AI 规划置信度

未实现原因：

当前前端右侧百分比显示的是 AI 返回的 `confidence`，不是浏览器语音识别的 confidence。不同浏览器对 Web Speech API 的置信度支持不稳定，因此暂未接入。

### 11. 移动端完整适配

未实现内容：

- 移动端触控编辑
- 移动端布局深度优化
- 小屏幕下完整画布操作体验

未实现原因：

当前 CSS 有基础响应式布局，但主要体验仍面向桌面浏览器。移动端触控编辑需要额外交互设计。

### 12. 自动化测试

未实现内容：

- 后端单元测试
- AI 响应解析测试
- 前端 Canvas 渲染测试
- 浏览器端端到端测试

未实现原因：

当前项目处于快速原型阶段，尚未建立测试框架。尤其前端 Canvas 渲染需要额外引入 Playwright 或截图对比测试。

### 13. 安全和生产部署能力

未实现内容：

- API key 安全托管
- 请求限流
- 访问鉴权
- CORS 精细配置
- 生产日志和监控

未实现原因：

当前项目面向本地开发和演示，后端只做了基础 CORS 允许和最小 ASGI 响应，没有生产级安全策略。

## 当前限制

- 必须配置可用的 AI API，否则无法执行绘图命令。
- AI 输出质量直接影响绘图质量。
- 复杂图形依赖 AI 生成合理的路径。
- 前端对象状态不会持久保存。
- 当前没有本地规则兜底。
- 当前没有完整图层、拖拽、节点编辑能力。

## 后续优先级建议

建议优先实现：

1. 对象点击选择和基础拖拽。
2. 项目保存/加载。
3. AI 计划预览确认。
4. 更完整的样式协议。
5. 自动化测试。

这些功能能显著提升项目从演示工具到可用绘图工具的完整度。
